# Чеклист раздела «Профиль» (MVP по видению PDF)

Источник: раздел «Профиль» в `docs/Видение_ANGELO_2026_03_26_для_разработчиков.pdf`. Состояние кода — на момент составления. Пути — от корня репозитория.

| Требование из видения | Реализация | Пробелы |
|----------------------|------------|---------|
| 1. Управление аккаунтом | `src/pages/Settings.tsx`: ссылка на профиль, выход | Нет смены email/пароля, привязки телефонов, удаления аккаунта |
| 2. Настройка подписки | Заглушка в API: `backend/app/api/auth.py` `SubscriptionResponse` plan `free` | Нет экрана подписки, оплаты, лимитов |
| 3. Ссылки для приглашения | `src/pages/InviteFlow.tsx`, `ROUTES.classic.invite` | Нет централизованного «мои инвайты» в настройках |
| 4. Управление уведомлениями | Иконка в Settings, без бэкенда | Нет push/email настроек и API |
| 5. Настройки приватности | `src/contexts/PrivacyVisibilityContext.tsx`, UI в Settings | Локально для прототипа; нет синхронизации с сервером и тонких прав по контентам |
| 6. Настройки отображения контента | Тема, язык в Settings | Нет опций «как показывать ленту/таймлайн» с бэкендом |
| 7. Настройки микрофона и др. | `src/ai/AiShell.tsx` — голосовой сценарий | Нет системных разрешений/настроек в разделе Профиль как в ТЗ |
| 8. Покупки в ателье | `src/pages/StorePage.tsx` | Заглушка; заказы и история покупок отсутствуют |

**Профиль члена семьи:** `src/pages/MyProfile.tsx`, `src/pages/EditMyProfile.tsx`, API `/api/profile/me` (`backend/app/api/profile.py`).

**Рекомендуемые следующие шаги после утверждения MVP:** вынести подписку и уведомления в отдельные экраны; смоделировать `UserSettings` в БД для приватности и локали; связать инвайты с приглашениями на бэкенде.
