## Что сделано на этапе (кратко)

- Зафиксирована **трассируемость** видения к репозиторию (матрица).
- Подготовлены **продуктово-технические заготовки** без полной смены IA в проде: спайк мессенджера, черновик Event, чеклист профиля.
- В коде (репозиторий): **нормализация публичных URL** для медиа и аватаров (`storage_urls`, использование в feed/profile/family/auth); **опциональная навигация под видение** через переменные `VITE_VISION_IA_NAV`, `VITE_VISION_IA_HOME` (см. корневой `.env.example`), хаб сообщений и альтернативный `BottomNav`.

Детали по коду и путям — в матрице; исчерпывающий diff см. в истории коммитов репозитория после внедрения пакета.

## Что сознательно не делалось (ждут заказчика / дизайна)

- Полная замена нижнего меню для всех пользователей без флага.
- Миграция **Publication → Event** и групповой мессенджер со звонками без утверждённой модели и стека.

## Следующие шаги (рекомендация)

1. Утвердить **информационную архитектуру** и первый экран после входа.
2. Утвердить **модель событий** по `event-model-draft.md`, затем проектировать миграции.
3. Выбрать стратегию **мессенджера** по `messenger-spike.md`.
4. Закрыть приоритетные пункты **профиля** по `profile-mvp-checklist.md`.
5. После макетов: **коллекции** (иерархия, лента), **сквозные сценарии** (событие ↔ чат ↔ коллекции).
