## Синхронизация Lovable и GitHub

- Проект в Lovable подключён к репозиторию **pyLexxDramma/amber-family-tree** (настройка в Lovable: Settings → Connectors → GitHub).
- Двусторонняя синхронизация включена: правки в Lovable попадают в GitHub, пуши в ветку `main` подхватываются в Lovable.
- Деплой на https://amber-family-tree.lovable.app/ выполняется из Lovable (Share → Publish → Update).
- Дальнейшая разработка может идти и в Lovable, и локально с пушами в GitHub; единый источник правок — ветка `main`.

## Что необходимо сделать

### Критично для соответствия ТЗ

1. **Авторизация** — реальная отправка кода на email/телефон (бэкенд + SMS/email-сервис или интеграция с Telegram по брифу) и проверка кода. Сейчас — только UI и мок.
2. **Фильтры ленты (экран 8)** — добавить: автор, участники, дата публикации, дата события (диапазоны), чекбоксы «избранное» и «непрочитанное»; применить фильтры к списку публикаций.
3. **Семья (экран 13)** — вкладки «Обо мне» / «Семья» и «Профили» / «Группы» по ТЗ; при необходимости — родственное отношение в списке (жена, сын и т.д.).
4. **Профиль контакта (экран 15)** — блоки «Публикации контакта», «Публикации, где контакт участник», «Медиа» с переходами на ленту с соответствующими фильтрами; напоминание заполнить ник, если пусто.
5. **Настройки (экран 22)** — разделы: изменить имя, изменить фото, «Выйти из аккаунта», «Условия / Политика» (документы).
6. **Приглашения (экран 19)** — действия: «Скопировать ещё раз», «Отменить» для исходящих; для входящих — «Принять» / «Отклонить» с переходом в профиль контакта и заполнением ника/родства.
7. **Управление местами (экран 21)** — отдельный экран «Места»: «занято X из Y», список людей, «Пригласить на место», «Освободить место» с подтверждением.
8. **Видимость публикации (экран 11)** — реальный выбор «Видна для» / «Исключить для» (группы или контакты), а не заглушка.

### Желательно / доработки

9. **Пустое состояние ленты** — при отсутствии публикаций: текст в духе «Создайте первую историю» + CTA «Создать» / «Пригласить» (по ТЗ п. 7).
10. **Онбординг шаг 2** — в ТЗ после профиля упомянут «шаг 2» — уточнить у заказчика и при необходимости добавить экран (например, первый запуск дерева/ленты).
11. **Режим выбора медиа в ленте (п. 7.1)** — кнопка «режим выбора медиа для дальнейших действий» (множественный выбор для действий) — не реализована.
12. **Локализация** — сейчас смесь EN/RU; по макету — русские подписи; привести к одному языку (или добавить переключение).
