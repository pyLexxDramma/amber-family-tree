# Заливка на хостинг Reg.ru

## 1. Сборка проекта

В корне проекта выполните:

```bash
npm install
npm run build
```

В папке `dist/` появятся готовые файлы для загрузки на сервер.

## 2. Что заливать на Reg.ru

Заливайте **содержимое** папки `dist/` (не саму папку) в корень сайта домена:

- через **Файловый менеджер** в панели Reg.ru — в каталог `public_html` (или тот, что указан в настройках домена как корень сайта);
- или по **FTP**: подключитесь к хостингу, откройте каталог сайта и загрузите туда все файлы и папки из `dist/`.

Должны оказаться в корне сайта:
- `index.html`
- папка `assets/` (JS и CSS)
- `.htaccess` (перенаправление маршрутов SPA на index.html)
- папки `prototype/`, `fonts/`, `demo/` (если были в `public/` и попали в `dist/`)

## 3. Проверка

Откройте в браузере ваш домен (например, `https://angelo-test.ru`). Должна открыться главная. Переходы по разделам (Лента, Дерево, Семья и т.д.) и обновление страницы (F5) должны работать благодаря `.htaccess`.

## 4. Если хостинг не поддерживает .htaccess

На некоторых тарифах Reg.ru может использоваться nginx. Тогда перенаправление на `index.html` нужно настроить в панели хостинга (раздел «Настройка сайта» / «Правила») или попросить поддержку Reg.ru настроить fallback на `index.html` для всех путей.

## 5. Автодеплой через GitHub Actions

При пуше в ветку `main` проект собирается и содержимое `dist/` заливается на хостинг по FTP.

**Настройка (один раз):**

1. GitHub → репозиторий → **Settings** → **Secrets and variables** → **Actions** → **New repository secret**.
2. Создай четыре секрета:

| Имя | Пример | Описание |
|-----|--------|----------|
| `FTP_HOST` | `ftp.angelo-test.ru` или `server279.hosting.reg.ru` | Хост FTP из панели Reg.ru |
| `FTP_USER` | твой логин FTP | Пользователь FTP |
| `FTP_PASSWORD` | пароль FTP | Пароль FTP |
| `FTP_REMOTE_PATH` | `/` или `/angelo-test.ru` | Путь к корню сайта на сервере (куда попадает index.html) |

Путь `FTP_REMOTE_PATH` смотри в ispmanager: при входе по FTP открывай папку, в которой лежит твой сайт (где сейчас index.html, assets/). Если файлы в корне FTP — укажи `/`. Если в подпапке (например, `angelo-test.ru`) — укажи `/angelo-test.ru`.

3. Закоммить и запушь `.github/workflows/deploy.yml`. После следующего `push` в `main` запустится сборка и деплой.
