## ТЗ: Создание публикации (История / Альбом)

### Цель
Сделать экран создания контента, который повторяет UX из референс‑видео `/_ref/create-publication.mp4`, но с русскими названиями, и работает на текущем backend API (FastAPI `/api/feed`, `/api/media/presign`).

### Термины
- **История**: публикация, собранная из блоков (текст/фото/видео/аудио/вставка/альбом/вложение/жизненный урок).
- **Альбом**: публикация, в которой основное содержимое — набор медиа (фото/видео), допускается описание.

### Экраны и шаги
Экран `Создать` включает 3 шага:
- **1. История**
- **2. Инфо**
- **3. Публикация**

Переключение шагов возможно кнопками «Назад/Следующий шаг». Переход вперёд запрещён, если нет минимального содержимого.

### Шаг 1: История (Story Builder)
#### Заголовок истории
- Режим редактирования: поле ввода + кнопки **✓ (сохранить)** и **✕ (отмена)**.
- Ограничение длины: **80 символов**, показывать счётчик `N / 80`.
- Режим просмотра: заголовок + кнопка **карандаш** для редактирования.

#### Блоки истории
- История состоит из списка блоков.
- Поддерживаемые блоки (все доступны):
  - **Текст**
  - **Фото**
  - **Видео**
  - **Аудио**
  - **Вставка** (URL)
  - **Альбом** (URL или медиа‑набор; в MVP допускается URL)
  - **Вложение** (файлы документов)
  - **Жизненный урок** (текстовый блок, как «вывод/мораль»)

#### Добавление блоков
- Кнопка **+** открывает модальное окно выбора блока.
- Внизу списка есть **второй +** для добавления дополнительных блоков после скролла.

#### Редактирование/удаление
- Текстовые блоки: редактируются через редактор «Добавить текст/Жизненный урок» и сохраняются в блок.
- Медиа‑блоки: показывают список файлов, статусы загрузки, ошибки, удаление файлов.
- Блок можно удалить целиком.

#### Редактор текста
- Модальный экран «Добавить текст» (и аналогично для «Жизненный урок»).
- В MVP панель форматирования визуальная (без реального rich‑text), текст сохраняется как plain text.
- Кнопка «Добавить в историю» активна только если текст не пустой.

### Шаг 2: Инфо
#### Участники
- Выбор участников из списка членов семьи (multi‑select).
- Выбранные отображаются чипами, можно снять.

#### Дата
- Режим **Точная дата** или **Период**.
- Ввод: год/месяц/день (в MVP как инпуты).
- В backend передаётся `event_date` (для периода берём From).
- Флаг «приблизительно» -> `event_date_approximate`.

#### Место
- Toggle «Отметить на карте» (в MVP без реальной карты, только UI).
- Поле поиска адреса (в MVP без геокодинга).
- Текстовое поле `place`.

#### Теги/соавторы/связанные истории
- Теги: список `#tag`, добавление/удаление.
- Соавторы: multi‑select из членов семьи -> `co_author_ids`.
- Связанные истории: поле ввода (в MVP без привязки).

#### Тема (обязательно)
- `topic_tag` обязателен для публикации.

### Шаг 3: Публикация
#### Доступ (Assigned Access)
Варианты:
- **Все участники**
- **Группы** (в MVP реализуется как выбор людей)
- **Отдельные люди**
- **Только я**

Маппинг:
- `Только я` -> `visible_for = [my_id]`
- `Группы/Отдельные люди` -> `visible_for = [my_id + выбранные]` (уникализировать)
- `Все участники` -> `visible_for = null` (семейная видимость по умолчанию)

#### Гостевая ссылка
- Toggle и TIP‑карточка как UI.
- В MVP не влияет на backend (нет поля).

#### Публикация
- Кнопка «Опубликовать обновления».
- После успешной публикации перейти на экран **деталей созданной публикации**.

### Маппинг Story/Album на backend
Backend endpoint: `POST /api/feed`.

Используем:
- `title`: заголовок истории/альбома
- `text`: склеенный текст из текстовых блоков (через пустую строку)
- `media_keys`: ключи загруженных файлов из медиа‑блоков
- `participant_ids`, `co_author_ids`: из шага Инфо
- `topic_tag`, `event_date`, `place`, `visible_for`

Тип публикации `type`:
- Если есть только один тип медиа и нет текста -> `photo`/`video`/`audio`/`document`
- Иначе -> `text`

### Ограничения MVP
- Нет настоящих групп (groups == выбор людей).
- Нет реальной карты/геокодинга.
- Нет полноценного rich‑text.
- Нет reorder блоков (можно добавить позже).
