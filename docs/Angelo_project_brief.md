# Проект Angelo — Briefing для разработчика

## Идея и проблема

Люди стареют и уходят — и вместе с ними навсегда теряются семейные истории, фотографии, воспоминания. Нет ни одного нормального инструмента, который бы решал эту проблему.

**Angelo** — это закрытая семейная социальная сеть. Не для публики, не для подписчиков — только для своей семьи. Место, где хранится всё: фото, видео, аудио, истории, семейное дерево. И это должно жить вечно — даже если кто-то забудет оплатить хостинг.

---

## Платформа

**Telegram Mini App** — веб-приложение внутри Telegram (React, WebView).

Важный архитектурный момент: приложение намеренно делается с абстракцией от Telegram SDK. Это значит, что если нужно — его можно перевести в отдельный PWA без переписывания кода. Страховка от любых проблем с Telegram.

---

## Как это должно выглядеть

Стиль — **magazine-style галерея**. Не лента соцсети, не Google Photos. Большие, красивые фотографии с подписями, как в семейном журнале. Уютно, тепло, с историями.

Клиент предоставляет дизайн-макеты (4 ключевых экрана) — вёрстка делается по ним.

---

## Функциональность — что входит в демо

### Доступ и роли
- Закрытая сеть: только по приглашению
- Две роли: **Family Admin** (создаёт пространство, управляет) и **Family Member** (участник)
- Регистрация через Telegram (автоидентификация)

### Загрузка контента
- Фото, видео, аудио с описаниями и датами
- Мультиселект файлов при загрузке
- Лимиты: фото до **20 MB**, видео до **500 MB** (~5-10 мин FHD), аудио до **100 MB** (~60-90 мин)
- Хранение в оригинальном качестве

### Gallery Mode
- Основной экран — визуальная галерея
- Большие фото + подписи (кто это, когда, что происходило)
- Красивая, приятная навигация

### Семейное дерево
- Визуализация дерева с интерактивностью
- Клик по человеку → его фото, видео, истории

### Таймлайн
- Хронологическая лента событий всей семьи
- Фильтрация по конкретному члену семьи
- Видно, что происходило в жизни разных людей в одно и то же время

### AI-функции (входят в демо)
- **Speech-to-text (Whisper)** — записал голосом → получил текст
- **Стилизация текста** — разговорная речь превращается в читаемый связный текст
- **AI-ассистент** — чат-бот внутри приложения, помогает навигировать (найти фото, перейти в нужный раздел)
- **Анимация фото** — базовое "оживление" старых снимков

## Что НЕ входит в демо (будущие версии)

- **Голосовой ассистент "Angelo"** — голосовые команды типа "Angelo, покажи фото Сергея за 2024 год"
- **Продвинутая генерация видео** из фотографий
- **Face recognition** — распознавание и сортировка по лицам
- **Параллельные таймлайны** — визуализация событий нескольких людей одновременно
- **Родительский контроль** — управление правами для несовершеннолетних
- **Push-уведомления и фоновые процессы**
- **Пакетное скачивание**

## Критически важная тема: Legacy Storage

Это одна из ключевых идей всего проекта — данные не должны пропасть никогда. Даже если не оплатили хостинг. Даже если сервер упал.

**Требует исследования в Фазе 1:**
- Blockchain-распределение данных
- P2P-синхронизация
- IPFS и децентрализованное хранение
- Шифрование + приватность при использовании публичных блокчейнов
- Восстановление данных если выжило только одно устройство

Это не просто техническая задача — это философия продукта. Семейная память должна существовать вечно.

## Тестовый контент для демо

Чтобы демо выглядело живо, нужен заготовленный контент:
- 5 членов семьи (3 поколения)
- 50-100 фотографий
- 10-20 видео
- 5-10 аудиозаписей
- Семейное дерево из 15-20 человек

## Открытые вопросы (контекст для понимания)

Клиент и команда ещё обсуждают ряд вещей — тебе важно знать об этом, чтобы понимать, что архитектура должна быть гибкой:

1. **Масштаб демо:** команда предлагает ограниченный пилот (~50 пользователей), клиент хочет протестировать на реальной аудитории до 1000 человек. Пока не решено.
2. **Хранилище:** клиент хочет РФ-серверы (требования compliance/безопасности). На пилоте — Supabase для скорости, потом миграция. Архитектура должна это учитывать.
3. **AI-сортировка:** клиент интересовался сортировкой по лицам (face recognition). Это дорого и сложно — в демо не входит, но надо иметь в виду.
4. **Родительский контроль:** клиент поднимал тему управления правами для детей. Тоже за пределами демо, но вопрос открыт.

## Фазы работы (по содержанию)

### Фаза 1 — Архитектура и исследование
- Техническая архитектура (диаграмма)
- Исследование legacy-хранилища (blockchain / P2P / IPFS)
- Анализ дизайн-макетов клиента + UX-рекомендации
- Customer Journey Map (роли: Family Admin, Family Member)
- ER-диаграмма базы данных
- Финальный выбор стека

**Deliverables:** отчёт по исследованию, архитектурная схема, анализ дизайна, обоснование стека

### Фаза 2 — Разработка демо
- Вёрстка экранов по дизайну клиента
- Frontend + Backend
- Авторизация (закрытый доступ, приглашения)
- Загрузка медиа с метаданными
- Gallery mode
- Семейное дерево (базовая интерактивность)
- Таймлайн с фильтрацией
- AI-фичи (Whisper, стилизация, ассистент, анимация фото)

**Deliverables:** рабочее демо-приложение, исходный код, документация
