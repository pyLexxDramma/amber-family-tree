2. Требования Angelo (MVP)

Оценка: 100 семей (~500 человек), медиахранилище ~3 TB.

Схема: 1 VM + Managed DB + Object Storage.

3. Что не учтено в Предложении

3.1. SSL/HTTPS

Приложение требует HTTPS (angelo-test.ru). В КП не указано:

  • Сертификат (Let's Encrypt / Yandex Certificate Manager)
  • Терминация SSL на Load Balancer или на VM

3.2. Бэкапы PostgreSQL

Требуем retention 7–14 дней. Нужно уточнить:

  • Включены ли автоматические бэкапы в Managed PostgreSQL?
  • Стоимость retention 7–14 дней

3.3. Мониторинг и алерты

В MVP нужны: 5xx, latency, диск > 80%, недоступность /api.

  • Monitoring (Yandex Monitoring) — входит ли в базовый тариф?
  • Алерты в Telegram/Email

3.4. SMTP / отправка писем

Приложение отправляет OTP-коды по email (вход по коду). В .env:

  • SMTP_HOST, SMTP_USERNAME, SMTP_PASSWORD — нужен отдельный сервис или Yandex предоставляет?

3.5. Поддержка

В сноске КП указано: тариф «Бизнес» — 3 050 ₽/мес + 5% от потребления. Нужно уточнить, включена ли поддержка в расчёт или это дополнительно.

3.6. Миграция данных

Перенос с Fornex:

  • Дамп PostgreSQL + восстановление
  • Копирование медиа в Object Storage (S3 API)
  • Нужна ли помощь в миграции или консультация?

4. Дополнительные вопросы для Яндекс

  • PostgreSQL: Достаточно ли 4 GB RAM для 100 семей, или лучше сразу 8 GB? Разница в стоимости?
  • PostgreSQL: Версия PostgreSQL в Managed Service? (Приложение рассчитано на 15.x)
  • Object Storage: S3-совместимый API? Endpoint для подключения из приложения (boto3)?
  • Object Storage: Публичный доступ к объектам через URL (presigned URLs) — поддерживается?
  • SSL: Рекомендуемый способ получения и обновления сертификата для angelo-test.ru?
  • Load Balancer: Как связать с VM и доменом? Нужна ли отдельная настройка для HTTPS?
  • Зона доступности: КП указывает на РФ. Нужно подтвердить, что данные размещаются в РФ.
  • Срок действия: КП от 18.04.2026. До какого числа актуальны цены?
