# Вход и профили

## Реальный API (бэкенд)

**URL:** https://angelo-test.ru (после `deploy-fornex.sh`)

### Свой вход / регистрация

Любой email + любой код → **новый пустой профиль**. Заполняете в «Редактировать профиль» и работаете.

### Эталонный профиль (тест на реальном API)

**Email:** `alina.fadeeva@angelo-demo.ru`  
**Код:** `000000` (или любой 4–6 цифр)

**Шаги:**
1. Ввести email → «Получить код»
2. Ввести `000000` → «Подтвердить»
3. Попадаете в **уже заполненный** профиль (Алина Фадеева, Москва, о себе и т.д.)
4. Лента заполнена ~25 публикациями с фото, ~20 участниками семьи
5. Проверяете работу на реальном API

Первый вход с этим email создаёт аккаунт с полным профилем и сидом (публикации, участники). Повторные входы — тот же профиль.

На **Fornex** после входа используется **реальный API** и демо-флаг в браузере **выключен** — лента и семья из **БД** (сид при `verify`). Локально если нужна мок-лента после этих же кредов: в `.env.local` задать `VITE_REFERENCE_DEMO_UI=true`.

Если на сервере лента «ломаная» или после экспериментов много дублей: войти эталонным пользователем, взять `access_token` из ответа `/auth/verify`, выполнить `POST /api/debug/seed-reference` с заголовком `Authorization: Bearer <token>` — БД этой семьи пересидируется (только для `alina.fadeeva@angelo-demo.ru`).
